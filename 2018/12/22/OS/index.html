多道程序

概念

多道程序设计技术是在计算机内存中同时存放几道相互独立的程序，使它们在管理程序控制下，相互穿插运行，两个或两个以上程序在计算机系统中同处于开始到结束之间的状态，这些程序共享计算机系统资源。与之相对应的是单到程序，即在计算机内存中只允许一个程序运行

特点
  1. 多道：即计算机内存中同时存放几道相互独立的程序
  2. 宏观上并行：同时进入系统的几道程序都处于运行过程中，即它们先后开始了各自的运行，但都未运行完毕
  3. 微观上串行：从微观上看，内存中的多道程序轮流地和或分时地占有CPU
优点
  1. 提高CPU的利用率：多个程序共享计算机资源，当某个程序等待I/O操作时，CPU可以执行其他程序
  2. 提高设备的利用率：多个程序共享系统的设备，使得系统设备的利用吕提高
  3. 提高系统的吞吐量：多道程序环境下，减少了程序的等待时间，提高了系统的吞吐量

死锁

概念

死锁是指两个或两个以上的进程在执行过程中，由于竞争资源或者由于彼此通信而造成的一种阻塞的现象，若无外力作用，它们都将无法推进下去。此时称系统处于死锁状态或系统产生了死锁，这些永远在互相等待的进程称为死锁进程

原因
  • 竞争资源引起死锁
    • 可剥夺性资源（CPU，内存）和非剥夺性资源（打印机，磁带机）
    • 竞争非剥夺性资源可导致死锁
    • 竞争临时性资源可导致死锁
  • 进程推进顺序不当引起死锁
条件
  • 互斥条件(资源的临界性)
  • 请求和保持条件
  • 不剥夺条件
  • 环路等待
处理方法

  1. 预防：破坏四个必要条件之一（有效但是资源利用率低）

    • 摒弃互斥条件：互斥是资源固有属性，无法避免

    • 摒弃请求和保持条件：资源的一次性分配（全分配，全释放）

      缺点：（1）延迟进程运行（2）资源严重浪费

    • 条件：新申请不能满足则释放已获得资源

      缺点：增加系统开销，且进程前端工作可能失效

    • 摒弃“环路”条件：资源的有序分配法（为资源编号，申请时需编号进行）

      缺点：(1)新增资源不便(原序号已排定)(2)用户不自由(3)资源与进程使用顺序不同造成浪费

  2. 避免：防止进入不安全状态（允许动态申请资源，银行家算法）

    • 安全状态是在“避免死锁”方法中的判断条件

    • 按照某种顺序并发进程都能达到的获得最大资源而顺序完成的序列为安全序列

      能找到安全序列的状态为安全状态

    • 银行家算法看后面

  3. 检测：检测到死锁再清除

    • 死锁定理：S为死锁状态的充分条件是当且仅当S状态的资源分配图是不可完全简化。
    • 找出既不阻塞又非独立结点，去掉所有分配边和请求边；继续化简，若能完全简化，则不会发生死锁，否则会死锁。
    • 检测时间：
      • 每当有资源请求时去检测， 毫无疑问越早发现越好， 但这种方法会占用昂贵的 CPU 时间
      • 每隔 k 分钟去检测一次 ， 或者当 CPU 的 使用率降到某一限值时去检测
    • 重视 CPU 使用率的原因在于如果死锁的进程数达到一 定数量 ， 就没有多少进程可以被执行 ， CPU 会经常被限 制 。

  4. 解除：系统中若出现死锁现象，将有大量的系统资 源被占用、被浪费，甚至可能导致系统的崩溃。 因此，一旦检测出死锁的存在，就应该尽快地采 取措施，解除死锁，恢复系统的运行

    • 剥夺资源
      • 把一个资源从一个进程手中抢过来，交给另一个 进程使用，等它用完之后，再还给原来的进程
      • 在资源剥夺过程中所造成的不良影响的大小，完 全取决于该资源自身的性质。实际上，死锁问题 正是由不可抢占资源所引起的，所以强行剥夺资 源必然会影响进程的正常运行
      • 因此只能选择那些相对而言容易剥夺的资源
    • 进程回退
      • 定期地把每个进程的状态信息保存在文件当中， 这样就得到一个文件序列，每个文件分别记载了 该进程在不同时刻的状态。
      • 进程的状态信息包括它的内存映象和它所占用的 资源的状态
      • 当系统检测到死锁时，先查明有哪些资源涉及， 然后把其中一个资源的拥有者（进程）回退到以 前的某个时刻（尚未拥有该资源），从而打破死锁。但该进程从那时刻开始的所有工作都丢失了
      • 代价高昂
    • 撤销进程
      • 撤消一个或多个处于死锁状态的进程
      • 先撤消一个死锁进程（或未死锁但占用了资源的进程），若其他死锁进程能够运行起来，则说明有效；否则继续撤消进程，直到死锁解除
      • 为了减少伤害程度，应尽可能地选择那些能安全地重新运行的进程，如编译进程；而不要去选择 那些无法安全地重新运行的进程，如对数据库的更新。

设备无关性(设备独立性)

概念

指应用程序独立于具体使用的物理设备；涉及逻辑设备和物理设备；

分配

逻辑设备表（LUT）：包含三列属性逻辑设备，物理设备，Dirver入口

分配流程：进程给出逻辑名——通过LUT得到物 理设备及 其driver入口

优点
  • 设备分配更灵活：逻辑设备和物理设备间可以是多—多的映射关系。 提高了物理设备的共享性，以及使用的灵活性。
    • 某逻辑名可对应这一类设备，提高均衡性与容错性。
    • 几个逻辑名可对应某一个设备，提高共享性
  • 易于实现I/O重定向。
    • 不变程序，只需改变LUT表的映射关系。
实现
  • 为了实现设备的独立性，应引入逻辑设备和物理设备两个概念。
  • 在应用程序中，使用逻辑设备名称来请求使用某类设备；而系统执行时，是使用物理设备名称。
  • 鉴于驱动程序是一个与硬件（或设备）紧密相关的软件，必须在驱动程序之上设置一层软件，称为设备独立性软件，以执行所有设备的公有操作、完成逻辑设备名到物理设备名的转换（为此应设置一张逻辑设备表）并向用户层（或文件层）软件提供统一接口，从而实现设备的独立性。

假脱机系统的组成

概念

当系统中引入了多道程序技术后，完全可以利用其中的一道程序，来模拟脱机输入时的外围控制机功能，把低速I/O设备上的数据传送到高速磁盘上。再用另一道程序，模拟脱机输出时外围控制机的功能，把数据从磁盘传送到低速输出设备上。这 样，便可在主机的直接控制下，实现以前的脱机输入、输出功能。 此时的外围操作与CPU对数据的处理同时进行，我们把这种在联机情况下实现的同时外围操作的技术，称为SPOOLing技术 ，或称假脱机技术。

优点
  • 提高了I/O的速度
  • 将独占设备改造为共享设备。
  • 实现了虚拟设备功能。
图示

各部件作用
  1. 磁盘缓冲区：在磁盘上开辟的一个存储空间，用于暂存用户程序的 输出数据；
  2. 打印缓冲区：为了缓和CPU和磁盘之间速度不匹配的矛盾，在内存中要 有一个打印输出缓冲区，用于暂存从磁盘缓冲区送来的数据；
  3. 假脱机管理进程和假脱机打印进程：由假脱机管理进程为每个要求打印的用户数据，建立一个假脱机文件， 并把它放入假脱机文件队列中，由假脱机打印进程，依次对队列中的文件进行打印。

程序的局部性原理

概念

程序的局部性原理是指程序在执行时呈现出局部性规律，即在一段时间内，整个程序的执行仅限于程序中的某一部分。相应地，执行所访问的存储空间也局限于某个内存区域。

表现：时间局部性和空间局部性
  • 时间局部性是指如果程序中的某条指令一旦执行，则不久之后该指令可能再次被执行；如果某数据被访问，则不久之后该数据可能再次被访问。
  • 空间局部性是指一旦程序访问了某个存储单元，则不久之后。其附近的存储单元也将被访问

影响缺页率的因素

缺页

就是CPU所需访问的页面不在内存，就需要将页面调入内存，如果内存已满，就执行相应的页面置换算法

缺页率

指的是访问页面失败次数除以进程页面访问总次数，设访问成功次数为S，访问失败次数为F，访问总次数A=S+F，缺页率f=F/A。

影响因素
  • 页面大小。页面划分比较大，则缺页率较低；反之，缺页率较高。
  • 进程分配的物理块数。所分配的物理块越多，缺页率越低；反之，越高。
  • 页面置换算法。算法的优劣决定了进程执行过程中缺页中断的次数，因此缺页率是衡量页面置换算法的重要指标。
  • 程序固有特性。程序本身的编制方法对缺页中断次数有影响，根据程序执行的局部性原理，程序编制的局部化程度越高相应执行时的缺页程度越低。

文件的逻辑结构和物理结构

逻辑结构：指一个文件在用户面前所呈现的形式.

  • 基本要求

    • 有助于提高对文件的检索速度
    • 结构应方便对文件进行修改
    • 降低文件存放在外存上的存储费用

  • 分类

    • 有结构文件：记录式文件
      • a类（1）定长记录（2）变长记录
      • b类（1）顺序文件，通常是定长记录（2）索引文件（3）索引顺序文件，顺序组织多个组，每组记录中的第一个记录设置一索引项
    • 无结构文件：流式文件，字节为单位，利用读/写指针进行访问

  • 顺序文件

    • 排列方式：

      • 串结构：记录通常是按存入时间的先后进行排序，各记录之 间的顺序与关键字无关。对串结构文件进行检索是比较费时的。
      • 顺序结构：由用户指定一个字段作为关键字，文件中的所有 记录就可以按关键字来排序。顺序结构文件可有更高的检索速度和效 率。

    • 优点

      • 适应用于对诸记录进行批量存取的场合；
      • 对顺序文件的存取效率是所有逻辑文件中最高的；
      • 也只有顺序文件才能被存储在磁带上，并能有效地工作。

    • 缺点

      • 在交互应用的场合，顺序文件所表现出来的性能就可能很差；
      • 如果想增加或删除一个记录，都比较困难。

    • 记录寻址

      • 隐式寻址: (1)定长：如果已知当前记录的逻辑地址，便很容易确定下一个记录的逻辑地址。

        ​ (2)变长：每次都需要从正在读（写）的记录中，读出该记录的长度。

      • 显式寻址：该方式可用于对定长记录的文件，实现直接或随机访问。

        • 通过文件中记录的位置

          • （1）定长记录文件：${A_i=i * L}$
          • （2）不定长记录文件: ${A_i = \sum_{j=0}^{i - 1}{L_j} }$

        • 利用关键字：用户必须指定一个字段作为关键字，通过指定的关键字来查找该记录。 当用户给出要检索记录的关键字时，系统将利用该关键字顺序地从第一个记 录开始，与每一个记录的关键字进行比较，直到找到匹配的记录

  • 索引文件

    • 按关键字建立索引：为变长记录文件建立一张索引表，对主文件中的每个记录，在索引表中设有一个相应表项，用于记录该记录的长度L，以及指向该记录的指针(指向 该记录在逻辑地址空间的首址)。由于索引表是按关键字排序的，是一个定长记录的顺序文件，这样就把对变长记录顺序文件的顺序检索，转变为对定长记录索引文件的随机检索，从而加快对记录检索的速度，能实现直接存取。
    • 具有多个索引表的索引文件：不同的用户，为了不同的目的，希望能按不同的属性（或不同的关键字） 来检索一条记录。为实现此要求，需要为顺序文件建立多个索引表，即为每一种可能成为检索条件的域（属性或关键字），都配置一张索引表
    • 优点：（1）提高了对文件的查找速度（2）插入和删除记录非常方便
    • 缺点：增加了存储开销

  • 索引顺序文件

    • 特征：保留了顺序文件的关键特征，即记录是按关键字的顺序组织起来的。 增加了两个新特征： ①引入了文件索引表，通过该表可以实 现对索引顺序文件的随机访问； ②另一个是增加了溢出文件，用它来记录新增加的、删除的和修改的记录。

  • 直接文件：可根据给定的关键字，直接获得指定记录的物理地址，即关键字本身就决定了记录的物理地址。这种由关键字到记录物理地址的转换，被称为键值转换。

  • 哈希文件：利用Hash函数(或称散列函数)，可将关键字转换为相应记录的地址。

物理结构：是指文件在物理存储上的存储形式。

  • 连续分配：每个文件分配一组相邻盘块

    • 优点：(1)简单(2)顺序访问容易且速度快，因为磁头移动距离小
    • 缺点：(1)要求连续空间，一段时间后需整理磁盘以消除外部碎片(2)文件不易于动态增长和删除

  • 链接分配：文件离散的分配于各盘块中

    • 优点：提高外村利用率，文件长度可变，易于增删

    • 缺点：只能顺序存取

    • 隐式链接：文件目录表有start块号，每块有下一块号；

      ​ 只能顺序访问，对随机访问效率低，可靠性差

    • 显示链接：用于链接的指针显式存放在内存的一张表中，查找在内存中进行；文件目录中相应目录项中的物理地址即第一盘块号；查找记录过程在内存中进行，大大提高速度

  • FAT技术：将物理磁盘分成多个逻辑磁盘（卷）每个卷划出一个单独区域存放自己的目录，FAT表和逻辑驱动字母

    • 表项有n位，则FAT表中至多有${2^n}$个表项

  • 索引分配：

    • 单级索引
      • 概念：为每个文件分配一个索引块
      • 链接分配问题：不能高效直接存取；FAT需占较大内存
      • 特点：（1）文件大时有利，小时浪费外存空间（2）文件大时索引块多，查找慢
      • 解决：索引太大时建立多级索引
    • 多级索引
      • 每个盘块存放盘块号个数为${P_i}$
      • 存放文件盘块号总数为${\sum P_i}$
      • 文件最大长度：总数 * 盘块大小
    • 混合分配方式（UNIX系统）
      • 一、二、多级索引合用
      • 设每块大小4K，一索引项占4B，则①直接地址：小文件（<40k）立刻读出②一次间址：4m③多次寻址：4g->4T

    • 时间片大小q的确定：太大则退化为FCFS；太小则系统开销大
    • 系统对响应时间的要求：T=nq，n为就绪队列中的进程数
  • 多级反馈队列调度
    • 长短作业兼顾，较好的响应时间；
    • 短作业一次完成，中作业周转时间部长，大型作业不会长期不处理

银行家算法

一种能避免死锁的调度算法

  • 核心：满足最大需求之后即可以偿还资源

  • 特点：（1）允许动态申请资源，可提高资源利用率

    ​ （2）要求事先说明最大资源要求，现实中很困难

  • 流程：

    • S1 某个客户提出贷款请求
    • S2 假设批准该请求，将得 到系统状态T；
    • S3 判断状态T是否安全， 如果安全，则批准该请求，转S1； 如果不安全，则不批准该请求，延期到以后处理， 转S1；

  • 例题

    • 时刻的安全序列：1 -> 3 -> 4 -> 2 -> 0

虚拟地址(请求分页中逻辑地址物理地址转换)

分页存储
  • 逻辑地址=页号+页内地址
  • 块号地址=块号+页内地址
  • 页号地址位数n，最大可有${2^n}$个页面
  • 页内地址位数m，每页最大为${2^n}$B的内存
  • 题目：页号2对应块号9，用户编程空间6个页面，每页1KB，内存为16KB，逻辑地址为 0A5C(H)，求物理地址

${2^6=64，2^{10}=1K，2^4=16}$，页号地址6位， 页内地址10位，块号地址4位；

0A5C(H)=0000 1010 0101 1100(B) 页号(前六位)：2，页内(后十位)：10 0101 1100(B)

对应块号地址9，物理地址：10 0110 0101 1100(B) = 265C（H）

页内置换算法

OPT(最佳置换)
  • 所淘汰的页面：永不使用的或者是下次访问时间间隔最大的
FIFO(先入先出)
  • 所淘汰的页面：先进入内存的页面
LRU(最近最久未使用)
  • 所淘汰的页面：上次访问时间间隔最长的

磁盘调度算法（寻道6.6.2）

先来先服务FCFS
  • 按照到达顺序寻道
最短寻道时间优先SSTF
  • 在众多寻道之中找距离当前磁道最近的，以此类推
扫描（SCAN）算法
  • 又名电梯算法，先从当前出发往磁道号增加的方向访问，到最大之后再往下扫
循环扫描（CSCAN）算法
  • 先从当前出发往磁道号增加方向访问，到最大之后再去最小的磁道号，然后再往上
NStepSCAN算法
  • 步SCAN算法是将磁盘请求队列分成若干个长度为N的子队列，磁盘调度将按 FCFS算法依次处理这些子队列。
  • 而每处理一个队列时又 是按SCAN算法，对一个队列处理完后，再处理其他队列。
  • 当正在处理某子队列时，如果又出现新的磁盘I/O请求， 便将新请求进程放入其他队列，这样就可避免出现粘着 现象。
FSCAN算法
  • 又名分步电梯算法
  • 实质上是N步SCAN算法的简化，即FSCAN只 将磁盘请求队列分成两个子队列。
  • 一个是由当前所有请求磁盘I/O的进程形成的队列，由磁盘调度按SCAN算法进行处理。
  • 在扫描期间，将新出现的所有请求磁盘I/O的进程，放入另 一个等待处理的请求队列。这样，所有的新请求都将被推迟到下一次扫描时处理
